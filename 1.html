Diferencias entre desarrollador front-end, back-end y full-stack


El diseño front-end se refiere a lo que el usuario ve. Como dice la palabra, es la parte frontal de la página web. Esta parte de diseño la controlaremos mediante código html, hojas de estilo (CSS) para optimizar la estética y JavaScript para proporcionar interactividad y dinamismo a la página.

El diseño back-end, como indica la palabra, se refiere a la parte que va detrás de lo que el usuario ve a simple vista. Para ello utilizaremos el lenguaje PHP para procesar la información que recibimos del usuario. Esta parte no se ejecuta en el ordenador del usuario, sino en un servidor.

El diseño full-stack abarca tanto el diseño front-end como el back-end.


Framework CSS


Un framework CSS es una biblioteca de estilos css genéricos que se puede usar para realizar un diseño web. La principal ventaja es que proporciona una forma relativamente fácil y rápida de realizar un diseño web. Como principal inconveniente es que al ser bibliotecas de estilos usadas de forma masiva, se corre el riesgo de perder originalidad en el diseño y optimización de código.

Dos de los frameworks CSS más utilizados son Bootstrap y Tailwind CSS.


Bootstrap


Bootstrap (https://getbootstrap.com) sigue siendo en la actualidad el framework CSS de código abierto más famoso y utilizado, cuya versión actual es la 5.2.2. Cuenta con una base muy amplia de usuarios y va incorporando rápidamente las novedades CSS que van surgiendo. Se basa en un sistema GRID que permite diseñar usando un grid predefinido de 12 columnas donde se debe plasmar el contenido. Contiene gran variedad de componentes prediseñados y clases CSS. Todo esto contribuye a realizar diseños web responsivos de forma más fácil e intuitiva.


Tailwind CSS


Tailwind CSS (https://tailwindcss.com) es un framework CSS de código abierto que está alcanzando en popularidad a Bootstrap (aunque si consultamos los datos de Github, vemos que Bootstrap sigue siendo más popular), cuya versión actual es la 3.2.1. Al igual que Bootstrap, utiliza un grid predefinido de 12 columnas para situar el contenido.

¿Cuáles son las principales diferencias de Tailwind con respecto a Bootstrap?

  • La principal diferencia es que prácticamente no contiene componentes prediseñados, sino una gran cantidad de clases CSS, que aportan la gran ventaja de permitir un diseño mucho más personalizable y flexible que con Bootstrap.
  • Esta gran ventaja conlleva a su vez el inconveniente que los nombres de clase serán bastante más largos y complicados de configurar que con Bootstrap.
  • Tailwind permite usar todas sus funcionalidades con la descarga de un único archivo js ó dirección CDN. Sin embargo, para hacer lo mismo en Bootstrap se necesita descargar 2 archivos (js y css) ó direcciones CDN

Qué es WCAG


WCAG (Web Content Accessibility Guidelines, https://www.w3.org/TR/WCAG21) ó Pautas de Accesibilidad al Contenido Web son una serie de recomendaciones publicadas por el World Wide Web Consortium (W3C, https://www.w3.org) orientadas a conseguir que el contenido web sea más accesible principalmente para personas con discapacidades.


Validación en XHMTL


Con la validación conseguimos verificar que se cumplen las directrices que marca el W3C acerca del lenguaje, con lo que mediante la validación habremos hecho todo lo posible para que la página web se vea bien en cualquier dispositivo. Esto proporciona un valor añadido a nuestro diseño web. Ya será responsabilidad de cada navegador el mostrarla conforme a los estándares del lenguaje.


Servidor web


En su significado más amplio, un servidor web es un ordenador de gran potencia, dotado de un hardware y un software adecuados, que está al servicio de otros dispositivos o personas que le solicitan información.

Qué es Apache


Apache es un servidor web de código abierto. Está desarrollado y mantenido por una comunidad de usuarios en torno a la Apache Software Foundation, que entre otras cosas permite que haya parches de seguridad regulares y actualizados con frecuencia. Actualmente es uno de los servidores web más utilizados debido a su seguridad y estabilidad.


Qué es XAMPP


Xampp (https://www.apachefriends.org/es/index.html) es un paquete de software libre, cuyo nombre es un acrónimo:

  • X, que indica que es multiplataforma
  • A, de Apache como servidor web
  • M, de MariaDB/MySQL como sistema de gestión de base de datos
  • PP, de PHP y Perl como lenguajes de programación.
